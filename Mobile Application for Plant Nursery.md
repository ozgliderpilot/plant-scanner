# Task for AI: Develop a Mobile Application for a Plant Nursery Operated by Volunteers at the Royal Botanic Gardens Melbourne

## Background

The volunteer community at the Royal Botanic Gardens Melbourne operates its own plant nursery. To manage plant inventory, they use a Microsoft Access database.

Currently, any change that occurs in the physical nursery—for example, a plant sale, plant death, or repotting—is first recorded on paper by different volunteers. Later, this information is manually entered into the database by one or more people. As a result, some information is lost, never recorded on paper, or never entered into the database.

The current processes that affect plant quantities and/or database records are:

### 1. Creating a Plant Record and Assigning an Accession Number

- A new plant record is created in the database.
- One accession number is assigned to a single batch of plants.
- A batch may contain anywhere from 1 to N individual plants.

### 2. Repotting Plants

- Plants are transferred from tubes into pots.
- Their status is changed to "Ready for Sale".

### 3. Requesting a Printed Plant Label

Each label contains:

- Accession number (encoded in the barcode)
- Plant name
- Plant group (e.g., shrub, herb, grass, etc.)
- Light requirements (e.g., full sun, semi-shade)
- Barcode containing the accession number

Label data is retrieved from the Microsoft Access database and printed using NiceLabel software. A dedicated label printer is connected to the computer via a physical cable.

### 4. Selling Plants

Plant sales reduce the available inventory.

### 5. Plant Death

Dead plants must be removed from inventory records.

---

## Project Goals

1. Maintain accurate and up-to-date inventory data.
2. Ensure that the database always reflects the correct number of plants available for sale.
3. Reduce duplicate manual data entry and paper-based record keeping.
4. Improve the efficiency and reliability of nursery operations.

---

## Proposed Solution

### Phase 1: Android Mobile Application

#### 1. Barcode Scanning

Develop an Android mobile application capable of scanning plant label barcodes.

A comparable application is BookBuddy, which scans barcodes and retrieves item information.

#### 2. Plant List Synchronization

The application should be able to download and maintain an up-to-date list of nursery plants from Google Sheets (subject to discussion).

Plant data may be exported from the Microsoft Access database into Google Sheets (subject to discussion).

#### 3. Available Actions

During Phase 1, the application will support a single action:

- Sale

During Phase 2, additional actions will be added:

- Create new accession number
- Request label printing
- Tube-to-pot transfer
- Record plant death

---

## Sale Workflow

After scanning a plant label barcode or entering accession number manually, the application should:

1. Retrieve plant information using the accession number.
2. Allow the user to enter:
3. Number of pots sold
4. Full retail price per pot
5. Discount amount (if applicable)
6. Calculate and display the total amount payable by the customer.
7. Save the transaction to a sales list.

The sales list should contain:

- Receipt number (automatically generated by the application; used to group purchases made by a single customer)
- Accession number
- Plant name
- Plant group (retrieved from the plant list)
- Unit price
- Discount
- Total purchase amount
- Date and time of sale

### Phase 1 Output

On command, the application should export the sales list to Google Sheets.

The exported data should include the transaction date.

### Phase 2 Output

Sales transactions should automatically update plant quantities in the central plant database, ensuring inventory records remain accurate and current.
